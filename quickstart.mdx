---
title: 'Quickstart'
description: 'Clone, configure, and run the python-mariadb-example app in a few steps.'
---

<Note>
  Make sure your MariaDB server is running and the `scott` database has been created before starting the app. See [Prerequisites](/prerequisites) for setup instructions.
</Note>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/darylldan/python-mariadb-example
    cd python-mariadb-example
    ```
  </Step>

  <Step title="Install dependencies">
    The project has a single runtime dependency — the official `mariadb` Python connector.

    ```bash
    pip install -r requirements.txt
    ```

    The contents of `requirements.txt`:

    ```text
    mariadb
    ```
  </Step>

  <Step title="Review the connection configuration">
    The database connection is configured at the top of `main.py`. By default it connects to `127.0.0.1:3306` using the `scott`/`tiger` credentials:

    ```python
    sql = MariaDBInstance(
        user="scott",
        password="tiger",
        database="scott",
    )
    ```

    `host` and `port` are optional parameters with defaults of `"127.0.0.1"` and `3306` respectively (defined in `sql/sql_connector.py`). If your MariaDB server runs on a different host or port, pass them explicitly:

    ```python
    sql = MariaDBInstance(
        user="scott",
        password="tiger",
        database="scott",
        host="127.0.0.1",
        port=3306,
    )
    ```

    <Warning>
      If the connection fails, the app calls `sys.exit(1)` immediately. Double-check that MariaDB is running and that the `scott` user and database exist before starting.
    </Warning>
  </Step>

  <Step title="Run the app">
    ```bash
    python main.py
    ```

    On startup the app connects to MariaDB, instantiates the controllers and UI, then drops you into the main menu.
  </Step>

  <Step title="Use the CLI menu">
    The interactive menu looks like this:

    ```text
    [1] Get all employees
    [2] Get employee by employee no.
    [3] Get all departments
    [4] Get employee count per department
    [5] Insert employee
    [6] Delete employee by employee no.
    [0] Exit
    ```

    Enter the number for the action you want and press Enter. Here is what each option does:

    | Option | Action |
    |---|---|
    | `1` | Fetches and prints all rows from the `emp` table |
    | `2` | Prompts for an employee number and prints that employee's record |
    | `3` | Fetches and prints all rows from the `dept` table |
    | `4` | Runs a `LEFT JOIN` and prints employee counts grouped by department |
    | `5` | Prompts for all employee fields and inserts a new row into `emp` |
    | `6` | Prompts for an employee number and deletes that row from `emp` |
    | `0` | Exits the loop and closes the database connection |

    When inserting an employee (option `5`), enter `n/a` for any optional field you want to leave `NULL`, or type `/cancel` at any prompt to abort the operation.
  </Step>
</Steps>
