`UserInterface` (defined in `view/ui.py`) is the only layer the user interacts with directly. It owns both controllers, runs a `while True` event loop, reads user input from stdin, and delegates to the appropriate controller method.

## Constructor

```python
class UserInterface():
    emp_ctrl: EmployeeController
    dept_ctrl: DepartmentController

    def __init__(self, emp_ctrl: EmployeeController, dept_ctrl: DepartmentController):
        self.emp_ctrl = emp_ctrl
        self.dept_ctrl = dept_ctrl
```

<ParamField path="emp_ctrl" type="EmployeeController" required>
  An initialized `EmployeeController` instance used for all employee-related operations.
</ParamField>

<ParamField path="dept_ctrl" type="DepartmentController" required>
  An initialized `DepartmentController` instance used for all department-related operations.
</ParamField>

## Main menu

`print_main_menu()` writes the following numbered menu to stdout:

```
[1] Get all employees
[2] Get employee by employee no.
[3] Get all departments
[4] Get employee count per department
[5] Insert employee
[6] Delete employee by employee no.
[0] Exit
```

---

## `start()` — the main event loop

`start()` is the application's main loop. It prints the menu, reads the user's integer choice, and dispatches to the correct method using a Python `match` statement. Entering `0` returns from the method, which causes `main.py` to call `sql.close()` and exit.

```python
def start(self):
    while True:
        self.print_main_menu()
        inp = int(input("Enter choice: "))

        match inp:
            case 1: self.print_all_employees()
            case 2: self.print_employee_by_no()
            case 3: self.print_all_departments()
            case 4: self.print_employee_count_per_department()
            case 5: 
                res = self.insert_employee()
                if not res:
                    print("Insertion cancelled.\n")
            case 6: self.delete_employee()
            case 0: return
            case _ : 
                print("Invalid input!")
                continue
```

## Display methods

These methods retrieve data from the controllers and print it using the model's `__str__` method.

### `print_all_employees()`

Calls `emp_ctrl.get_all_employees()` and prints each `Employee` (the controller already calls `print(r)` internally, so this is displayed during the fetch).

### `print_employee_by_no()`

Prompts for an employee number with a plain `input()` call, fetches the record, and prints it. Displays `"Employee not found!"` if the controller returns `None`.

```python
def print_employee_by_no(self):
    empno = int(input("Enter employee number: "))

    result = self.emp_ctrl.get_employee_by_id(empno)

    if result == None:
        print(f"Employee not found!\n")
        return
    
    print(result)
```

### `print_all_departments()`

Calls `dept_ctrl.get_all_dept()` and prints each `Department` object.

### `print_employee_count_per_department()`

Prints a two-column table with department name and employee count, using the tuple results from `dept_ctrl.get_employee_count_per_department()`.

```python
def print_employee_count_per_department(self):
    print(f"Department Name\t\tEmployee Count")

    for r in self.dept_ctrl.get_employee_count_per_department():
        print(f"{r[0]}\t\t{r[1]}")
```

## `get_user_input(prompt, input_type)` — validated input helper

All multi-field operations use `get_user_input` instead of calling `input()` directly. It loops until the user provides a valid value, types `/cancel`, or enters `n/a`.

```python
def get_user_input(self, prompt: str, input_type: Literal["str", "int", "float", "date"]):
    while True:
        user_input = input(prompt + "\nEnter /cancel to abort operation\nInput: ")

        if user_input == "/cancel":
            return "__ABORT__"
        
        if user_input == "":
            print(f"Please enter 'n/a' for empty field.\n")
            continue

        if user_input == "n/a":
            return None

        if input_type == "str":
            return user_input
        
        if input_type == "int":
            try:
                converted = int(user_input)
                return converted
            except ValueError:
                print("Invalid ingeter. Please try again.")
                continue
        
        if input_type == "float":
            try:
                converted = float(user_input)
                return converted
            except ValueError:
                print("Invalid float. Please try again.")
                continue
        
        if input_type == "date":
            try:
                converted = datetime.strptime(user_input, "%Y-%m-%d")
                return converted
            except ValueError:
                print("Invalid date. Please try again.")
                continue
```

### Input type validation

<ParamField path="input_type" type="string" required>
  Controls how the raw string is parsed before being returned.

  - `"str"` — returned as-is, no conversion.
  - `"int"` — converted with `int()`. Re-prompts if `ValueError` is raised.
  - `"float"` — converted with `float()`. Re-prompts if `ValueError` is raised.
  - `"date"` — parsed with `datetime.strptime(value, "%Y-%m-%d")`. Re-prompts if the format does not match.
</ParamField>

### Special inputs

| Input | Behaviour |
|---|---|
| `/cancel` | Returns the sentinel string `"__ABORT__"`, signalling the caller to abort the operation. |
| `n/a` | Returns `None`, which maps to a `NULL` value in the database for nullable fields. |
| *(empty string)* | Rejected with a prompt to use `n/a` instead, then re-prompts. |

<Tip>
  Use `n/a` to leave a nullable field empty (e.g., `comm` for an employee with no commission). Pressing Enter without typing anything is not accepted — this distinction prevents accidental blank submissions.
</Tip>

## `insert_employee()` — multi-step input flow

Collects all eight employee fields one at a time via `get_user_input`, checks for `/cancel` after each field, constructs an `Employee` dataclass, and calls `emp_ctrl.insert_employee()`.

```python
def insert_employee(self) -> bool:
    empno = self.get_user_input("Enter employee number:", "int")
    if empno == "__ABORT__":
        return False

    ename = self.get_user_input("Enter employee name:", "str")
    if ename == "__ABORT__":
        return False
    
    job = self.get_user_input("Enter job:", "str")
    if job == "__ABORT__":
        return False
    
    mgr = self.get_user_input("Enter manager's employee no.: ", "int")
    if mgr == "__ABORT__":
        return False
    
    hiredate = self.get_user_input("Enter hiredate (YYYY-MM-DD):", "str")  # "str" — no format validation
    if hiredate == "__ABORT__":
        return False
    
    sal = self.get_user_input("Enter salary:", "float")
    if sal == "__ABORT__":
        return False
    
    comm = self.get_user_input("Enter commission:", "float")
    if comm == "__ABORT__":
        return False
    
    deptno = self.get_user_input("Enter Department No.:", "int")
    if deptno == "__ABORT__":
        return False
    
    new_emp = Employee(
        empno=empno,
        ename=ename,
        job=job,
        mgr=mgr,
        hiredate=hiredate,
        sal=sal,
        comm=comm,
        deptno=deptno
    )

    res = self.emp_ctrl.insert_employee(new_emp)
    if res == None:
        print(f"Failed to insert.\n")
        return
    
    print(f"Successfully inserted employee:")
    print(new_emp)
    return True
```

If the user types `/cancel` at any prompt, `get_user_input` returns `"__ABORT__"`, the method returns `False`, and `start()` prints `"Insertion cancelled."`.

**Returns:** `bool` — `True` on successful insert, `False` if cancelled.

## `delete_employee()` — existence check before deletion

Before deleting, `delete_employee()` fetches the employee by number to confirm they exist. If the record is not found it aborts early. This avoids issuing a `DELETE` against a nonexistent row and gives the user a clear error message.

```python
def delete_employee(self):
    empno = self.get_user_input("Enter employee number to delete:", "int")

    emp = self.emp_ctrl.get_employee_by_id(empno)
    if not emp:
        print(f"Employee not found.\n")
        return
    
    res = self.emp_ctrl.delete_employee_by_empno(empno)

    if res == None:
        print(f"Deletion failed\n")
        return
    
    print(f"Successfully deleted {emp.ename}.")
```

<Note>
  The cancellation mechanism (`/cancel`) is available during the employee-number prompt in `delete_employee()` because it calls `get_user_input`. The subsequent confirmation and deletion steps do not prompt again, so there is no further opportunity to cancel once the employee number is accepted.
</Note>
